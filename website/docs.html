System Protocols

Cognito is a quiet sanctuary for neural exploration. This guide details the essential architecture and operational patterns required for absolute local autonomy.

Foundation

The system is built on a trinity of specialized layers, designed for modularity and silent performance:

  • Interface Layer (React/Electron): A serene, gesture-focused shell that manages the visual state and system bridge.
  • Inference Layer (FastAPI): A high-performance localized service that maps neural requests to llama.cpp.
  • Context Layer (RAG Engine): A sparse retrieval system that allows the model to "remember" and "sense" your private documents.

Preparation

Deployment is a ritual of setting up a clean, isolated environment. We recommend source-based builds for total visibility.

Prerequisites

  • Node.js (v18+): The engine for our digital interface.
  • Python (v3.10+): The core processing runtime.
  • C++ Compiler: Essential for binding the neural engine to your specific hardware (Metal/CUDA).

Knowledge Core

The RAG (Retrieval-Augmented Generation) engine allows for seamless document integration without cloud dependencies.

Wabi-Sabi Retrieval: Our TF-IDF engine values simplicity and speed. It maps your documents to an in-memory vector space, ensuring zero-trace persistence after the sanctuary is closed.

Default parameters focus on balanced digestion: 400-word segments with soft 50-word overlaps to maintain narrative continuity.

External Senses

Cognito features an Agentic Search capability. This is not a manual tool, but a sensory organ that the model uses when it identifies a gap in its internal weights.

// The model triggers a sense check:
[SEARCH: specific inquiry]

The system then brings back mists of external data, which are distilled and presented to the model for final reasoning.

Neural Weights

We embrace the GGUF format. For a harmonious experience, we recommend 4-bit or 5-bit quantization (Q4_K_M), which offers high intelligence with a gentle footprint on system memory.
